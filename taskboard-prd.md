## 一、产品定位

**一句话定义：** 让有需求的人快速发布任务，让有技能的人快速接单，完成从发布到付款的全闭环。

**App 名称：** Taskly（英文）/ 活儿（中文）
**目标市场：** 海外优先（欧美 / 澳洲 / 东南亚）→ 国内二期
**主语言：** 英文优先，中文后续

**核心差异（vs 58同城 / 美团）：**
- 双向发布：需求方发任务 + 技能方发服务，两者互相可见
- 任务闭环：发布 → 匹配 → 沟通 → 付款托管 → 确认完成 → 评价
- 移动优先：轻量、快速、卡片化 UI，适合蓝领用户低门槛使用
- 资金安全：平台托管，完成才放款，保护双方利益

## 四、核心用户流程

### 流程 A：发任务方
```
注册/登录
  → 选任务类型（维修 / 搬运 / 跑腿 / IT / 其他）
  → 填写任务详情（标题 + 描述 + 图片 + 地址 + 预算 + 期望时间）
  → 发布任务（全平台可见，依赖自然流量匹配）
  → 收到系统推送：「附近有X人可接此任务」
  → 查看接单者主页 + 评分历史
  → 选择接单者 / 等待对方主动抢单
  → 预付款进入平台托管账户（资金不直接给工人）
  → 任务完成 → 确认放款
  → 双向评价
```

### 流程 B：接单方
```
注册/登录
  → 填写技能标签 + 服务城市/区域
  → 发布自己的服务卡片（可选）
  → 浏览任务广场（全量，不限城市）
  → 一键抢单 / 发送报价
  → 双方确认 → 开始任务
  → 完成后提交完工照片
  → 等待发单方确认（超时自动放款）
  → 收款（早期免佣金，后期扣除 10%）
```

## 五、功能清单

### MVP 阶段（上线必须有）

#### 账户体系
- [ ] 手机号注册/登录（国内：短信验证码）
- [ ] 邮箱注册/登录（海外）
- [ ] Apple / Google 第三方登录
- [ ] 单账号双角色：发单方 + 接单方自由切换
- [ ] 头像 + 简介 + 技能标签
- [ ] 接单方实名认证（MVP 阶段**强制**，上传身份证/护照，后台人工审核，未认证不可接单）

#### 任务模块
- [ ] 发布任务：类型 / 标题 / 描述 / 图片（最多6张）/ 地址 / 预算 / 截止时间
- [ ] 发布服务卡片：技能方主动展示自己（技能 + 价格 + 服务范围）
- [ ] 任务广场：全量展示，支持按类型 / 距离 / 价格筛选
- [ ] 任务详情页
- [ ] 抢单 / 报价功能（接单方可发起议价）
- [ ] 任务状态流转：待接单 → 进行中 → 待确认 → 已完成 / 已取消

#### 匹配与沟通
- [ ] 站内聊天（文字 + 图片）
- [ ] 系统通知推送（新抢单 / 任务更新 / 付款提醒）
- [ ] 发任务后可查看并主动邀请附近技能者

#### 支付模块（重点，保障平台安全）
- [ ] **MVP 优先**：Apple Pay（国内 + 海外统一方案，iOS 审核友好）
- [ ] **国内后续**：支付宝 + 微信支付（二期补充）
- [ ] **海外**：Stripe（支持 USD / AUD / SGD 等多币种，Apple Pay 底层即 Stripe）
- [ ] **资金托管机制**：
  - 发单方付款 → 资金冻结在平台托管账户
  - 接单方完成 → 提交完工照片申请放款
  - 发单方确认 → 平台释放资金给接单方
  - 发单方 48 小时未确认 → 系统自动放款（防止恶意卡款）
  - 有纠纷 → 进入人工仲裁流程
- [ ] 退款规则：
  - 任务未开始取消：全额退款
  - 任务进行中取消：双方协商，平台介入仲裁
- [ ] 接单方提现（T+1 到账）
- [ ] 佣金策略：**早期 0 佣金**（系统保留字段，随时可开启）

#### 信任与安全
- [ ] 双向评价（5星 + 文字 + 图片）
- [ ] 累计评分 + 完成单数展示
- [ ] 举报 + 拉黑功能
- [ ] 完工照片确认机制
- [ ] 平台客服入口（纠纷处理）
- [ ] 异常交易风控（金额异常 / 账号异常）

### 二期功能（上线后迭代）
- AI 匹配推送：发任务自动通知附近最匹配的人
- 技能认证徽章：上传资质证书，平台审核后显示认证标志
- 信用分体系（影响排序权重）
- 任务模板（常见任务一键发布，降低发布门槛）
- 多语言切换（中文 / 英文 / 马来文）
- 接单方收入数据面板
- 10% 佣金正式开启

## 七、信息架构（App 页面结构）

```
Tab 1：首页（任务广场）
  ├── 顶部：「发布任务」按钮（醒目，常驻）
  ├── 分段控制：任务列表 / 服务列表
  ├── 筛选栏：类型 / 距离 / 价格 / 最新
  └── 卡片列表（任务卡：标题/预算/地址/时间/剩余名额）

Tab 2：地图
  └── 附近任务 + 附近技能者分布（地图 pin）

Tab 3：消息
  ├── 系统通知（抢单/支付/完成提醒）
  └── 聊天列表

Tab 4：我的
  ├── 我发布的任务（状态追踪）
  ├── 我接的任务（进度追踪）
  ├── 钱包（余额 / 托管中 / 提现 / 交易记录）
  ├── 我的评价 + 评分
  └── 设置（语言切换 / 实名认证 / 帮助中心 / 关于）
```

## 九、技术方案（海外优先，单人版）

| 层 | 方案 | 说明 |
|----|------|------|
| iOS（先行）| Swift + SwiftUI | 原生，审核友好 |
| Android | 三期 | 先跑通 iOS |
| 后端 | **Supabase** | Auth + DB + Storage + Realtime 一体，单人首选 |
| 即时通讯 | **Supabase Realtime** | WebSocket 简版，文字 + 图片，够 MVP |
| 地图 | **MapKit**（海外优先）| 免费，原生；高德二期国内版再加 |
| 支付 | **Stripe + Apple Pay** | Apple Pay 走 Stripe 通道，绕过苹果抽佣 |
| 推送 | APNs | Supabase Edge Function 触发 |
| 图片存储 | **Supabase Storage** | 统一平台，省运维 |
| 实名认证 | 早期人工审核 → 后期 Stripe Identity | 先控成本 |
| 风控 | Stripe Radar（支付风控）+ 简单规则 | 不自建 |

## 十、团队与里程碑

**团队：** 1 人（独立开发）

### 单人开发优先级原则
- 能用 SaaS/SDK 替代的，不自建
- 能配置的，不写代码
- 先跑通核心闭环，其他功能全部砍掉或延后

### 技术栈调整（单人友好版）

| 层 | 方案 | 为什么 |
|----|------|--------|
| iOS | Swift + SwiftUI | 原生，审核友好 |
| 后端 | Supabase | 省掉自建 Auth / DB / Storage / Realtime，1人够用 |
| 即时通讯 | Supabase Realtime（WebSocket）| 无需另建服务，简版够 MVP |
| 地图 | MapKit | 免费，iOS 原生 |
| 支付 | Stripe + Apple Pay | Stripe SDK 直接支持 Apple Pay，一套搞定 |
| 推送 | APNs + Supabase Edge Function 触发 | |
| 图片存储 | Supabase Storage | 统一平台，省运维 |
| 实名认证 | 手动审核（早期）→ Stripe Identity（后期）| 先人工，量大再自动化 |

### 里程碑（单人版，20 周）

| 阶段 | 周期 | 目标 | 砍掉什么 |
|------|------|------|---------|
| 阶段一 | 第 1-5 周 | 账户 + 双向发布 + 任务广场 + 基础筛选 | 地图视图延后 |
| 阶段二 | 第 6-10 周 | 站内聊天 + 抢单流程 + Stripe 支付托管 | 风控延后 |
| 阶段三 | 第 11-15 周 | 完工确认 + 评价 + 推送 + 实名审核后台 | AI 匹配延后 |
| 内测 | 第 16-18 周 | TestFlight 发布，找 20 个真实用户测试 | |
| 上线 | 第 19-20 周 | App Store 审核提交 | |
